# Technical Report: N CUBE & Proof-of-Decision Protocol

## Executive Summary
This report outlines the implementation and integration of the **Proof-of-Decision Protocol (PDP)** within the **N CUBE** ecosystem. By leveraging **WeilChain's** WADK SDK and the **Model Context Protocol (MCP)**, N CUBE provides an immutable, verifiable audit trail for AI operations, bridging the gap between autonomous AI logic and human accountability.

## 1. Technological Innovations

### 1.1 Model Context Protocol (MCP) Integration
N CUBE implements a **FastMCP** server that allows AI models to directly interact with the PDP. This enables:
- **`submit_decision`**: Atomically records AI reasoning and triggers an on-chain audit.
- **`approve_decision/reject_decision`**: Facilitates human-in-the-loop governance.
- **Verification**: Tools to check the status of any decision against the blockchain.

### 1.2 WADK SDK & WeilChain
The core of N CUBE's trust layer is the **Weilliptic ADK**. 
- **On-Chain Logs**: Each decision is hashed and stored via `WeilClient.audit()`.
- **Wallet Auth**: The system uses `PrivateKey.from_file('private_key.wc')` to sign transactions.
- **Sentinel Integration**: Interacts with the Weil Sentinel for high-availability chain access.

### 1.3 Modern Full-Stack Architecture
- **Async Python**: Utilizing `FastAPI` and `uvicorn` for high-concurrency decision processing.
- **Premium Frontend**: A Next.js 14 dashboard using `framer-motion` for a state-of-the-art user experience.

## 2. System Workflow

1. **AI Interaction**: An AI agent calls the `submit_decision` tool via MCP.
2. **Local Storage**: The decision is stored in an async SQLite database.
3. **On-Chain Audit**: The `WeilService` generates a cryptographic hash and submits it to WeilChain.
4. **Human Review**: A human reviewer sees the pending decision on the dashboard.
5. **Review Audit**: Approval or rejection is also logged on-chain, creating a complete provenance record.

## 3. Future Roadmap
- **Advanced ZK Proofs**: Integrating zero-knowledge proofs for private reasoning traces.
- **Decentralized Reviewers**: Scaling the review process across multiple wallet owners.
- **Automated Slashing**: Implementing economic incentives for honest AI reporting.
